linux设备种类

  1. 字符设备: 按字节流的形式进行数据读写，一般情况下按顺序访问，数据量不大，一般不设缓存.

  2. 块设备: 按整块进行数据读写的设备，最小的块大小为512字节（一个扇区），块的大小必须是扇区的整数倍，Linux系统的块大小一般设为4097字节，随机访问，设缓存以提高效率.

  3. 网络设备: 针对网络数据收发设备.

设备号

内核用设备号区分同类里面的不同设备，设备号是一个32位的无符号整数，数据类型为dev_t，分为两个部分:

  1. 主设备号：占高12位，用来表示

  2. 次设备号：占低12位，用来表示被操作的哪个具体设备

  2. MAJOR()

  3. MINOR()

（一）内核模块

向内核新增功能

  1. 静态加载法

    即新功能源码与linux内核源码在同一目录结构下，然后一起编译进linux

    • a. 在驱动目录(例如: /driver/char/)下编写驱动代码(myhello.c)

    • b. 给新功能配置Kconfig

    • c. 给新功能代码改写Makefile

    • d.在顶层目录执行 make menuconfig 将新功能选项选成<*>

    这样，驱动就被加入到了内核中，下次make uimage之后，驱动就一并被编译.

  2. 动态加载

    a. 新功能源码与linux内核源码在同一目录结构下

    • 1 给新功能配置Kconfig
    • 2 配置Makefile
    • 3 make menuconfig 选择成< M >
    • 4 make uimage
    • 5 最后make modules，会在新功能源码目录下生成同名的.ko文件

    b. 新功能源码与linux源码不在同一目录结构下

    • 1 在对应目录编写驱动代码
    • 2 vim Makefile 并改写对应 Makefile
    • 3 make

    c. 主机使用ko文件

    d. 开发板linux使用ko文件

内核代码基础模块解析

  1. __init 的作用：一个宏，展开后为 “attribute ((section (“.init.text”)))”. 在模块插入时方便内核从ko文件指定的位置读取该入口函数读到内存中去.

  2. __exit 的作用: “attribute ((section (“.exit.text”)))”，类似__init

  3. MODULE_LICENSE(内容): 内容为源码的许可证协议

  4. module_init(入口函数)

  5. module_exit(出口函数)

什么是spec文件?

sepc文件是辅助构建rpm包的一种文件格式，里面指明了构建软件包需要的依赖信息.

一、常见参数说明(会一直更新)

（一）Preamble Items

  1. Name

    软件包的名称: 可以用%{name}引用.

  2. Version

    软件包的版本号: 可以用%{version}引用.

  3. Release

    软件包的发行号: 可以用%{release}引用.

Name，Version，Release组成了rpm包的名字:

The Name, Version, and Release directives comprise the file name of the RPM package.

  1. Summary

    对软件包的简单描述，这里随意写.

  2. License

    授权方式，不同的软件可能不同，常见的是GPL.

  3. URL

    上游网站的链接.

  4. Source0/1/2/…

    常常和URL搭配，这里也可以写软件包的名字，这里举个例子.

  5. BuildRequires

    这个指令指定了构建该软件包所需的其他软件包。这些软件包通常包含编译器、开发库、构建工具等。在构建软件包时，系统会检查并确保这些依赖软件包已经安装，以便顺利完成构建过程。通常，BuildRequires 指令在 RPM 包的 SPEC 文件的 %build 部分中使用.

  6. Requires

    这个指令指定了运行该软件包所需的其他软件包。这些软件包通常包含运行时库、依赖的系统工具等。在安装软件包时，系统会检查并确保这些依赖软件包已经安装，以便软件包能够正常运行。通常，Requires 指令在 RPM 包的 SPEC 文件的 %install 或 %post 部分中使用.

  7. BuildArch

    软件包依赖的架构，不依赖任何架构就写noarch

  8. Patch0/1/2/…

    补丁文件.

（二）Body Items

  1. %description

    RPM包的完整描述

  2. %prep(预处理)

    通常用来执行一些解开源程序包的命令，例如：unpack source0的内容，除了可以执行RPM所定义的宏命令（以%开头）以外，还可以执行shell命令，一般包含%setup和%patch两个命令.

  3. %build(编译)

    定义编译软件包所要执行的命令，如果要编译的宏使用了 autoconf，那么应当用 %configure 宏来配置软件.

  4. %setup

    参数详解: https://rpm-packaging-guide.github.io/#setup

    解压源代码，将当前目录改为源代码解压之后产生的目录.

    选项:

    • -q 在安静silence模式下执行，即在执行时不会显示解压缩的详细信息.
    • -n 用于指定解压后的目录名，默认情况下，解压后的目录名将与压缩包的文件名相同，只是去掉了扩展名.
    • -D and -T
      • -D disables deleting of source code directory
      • -T disables expansion of the source code tarball by removing the following line from the script.
    • -a and -b
      • -a after
      • -b before
    • -c 创建一个目录，并在目录里执行

  5. %patch

    • %patchN 表示使用第N个补丁文件

  6. %install

    安装软件包时候执行命令，类似于make install.

  7. %clean

    rm-rf $RPM_BUILD_ROOT

  8. %files

    列出所有将会被安装到最终用户系统中的文件。这些文件的路径应该相对于 %buildroot 目录，即安装后文件在系统中的路径.

  9. %changelog

    更新日志
